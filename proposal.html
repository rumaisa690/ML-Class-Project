Project Proposal

Introduction/Background

For generations, asteroid identification has been foundational in astronomical research. The classification of asteroids, particularly distinguishing those deemed hazardous, holds significant importance not only in protecting our planet from potential threats but also in deepening our comprehension of the universe. Our project aims to develop a model that classifies asteroids as hazardous or non-hazardous using data pre-processing methods and supervised learning algorithms. Utilizing data from the NASA API, our dataset includes features like absolute magnitude and relative velocity to develop a robust model that automatically identifies hazardous asteroids.

Problem Definition

With thousands of asteroids traversing our universe, the need for efficient methods to identify hazardous objects is important in supporting the astronomical field in research, exploration, and real-time monitoring. Understanding the risks posed by asteroids enables scientists to develop strategies to mitigate potential impacts, protecting life and infrastructure. Moreover, researching the characteristics of hazardous asteroids provides insights into their composition, behavior, and evolution, which contribute to broadening our understanding of space [2]. Furthermore, real time monitoring of these hazardous asteroids can help scientists to track the object's trajectory and evaluate potential impacts on Earth or satellites. Our project aims to use machine learning techniques to identify hazardous asteroids, protecting the Earth and supporting astronomical research.

Methods
  • 3+ Data pre-processing methods
    • Feature Selection
      • Identifies most relevant features contributing to classifying a hazardous asteroid
      • Reduces redundant/irrelevant features
      • Reduces chances of overfitting
    • Time Series Analysis
      • Categorizes dataset into components on attributes
      • Identifies temporal patterns to discover trends
      • Ex: Whether specific time periods associate with increased asteroid activity
    • Oversampling
      • Deals with dataset imbalance
      • Generates synthetic minority class instances to enhance model training
      • SMOTE for reducing bias, prediction reliability
    • Feature Scaling
      • Scaling to a similar range helps with model performance
      • without, ML models will prioritize larger features & skew data
  • Identify 3+ ML algorithms/models
    • All are supervised learning methods
      • Random Forest
        • Constructs decision trees during training & outputs class mode or mean prediction of trees
        • Gives feature importance & influential attributes
        • sklearn.ensemble
      • Support Vector Machines
        • Effective with high-dimensional feature spaces
        • Have been used in asteroid dynamics before [1]
        • Maximizes margin between different classes, prevents overfitting
        • sklearn.svm
      • Logistic Regression
        • Focuses on binary classification tasks
        • Puts asteroids into categories
        • Coefficients for each attribute to determine influence
        • sklearn.linear_model
      • Naive Bayes
        • Assumes independence among the many features in dataset [3]
        • Models conditional probability of each feature, robust to irrelevance
        • ‘MultinomialNB’ from sklearn.naive_bayes
Potential Results and Discussion
  • 3+ Quantitative metrics
    • The F1 score
        • useful for imbalanced classes
        • Higher scores = better model performance
    • Logarithmic loss
        • confident incorrect predicts
        • model calibration and discrimination ability
    • The Area Under the Curve (AUC)
        • quantifies classifier performance across thresholds, assess discrimination ability.
  • Project goals
    • Develop an efficient model for classifying hazardous asteroids, improving real-time monitoring
    • Impact astronomy by automating asteroid identification, streamlining researchers' work.
  • Expected results
    • High accuracy in classifying asteroids with our chosen ML models
    • Techniques like feature selection enhance identification, while naive bayes disregards irrelevant features
    • Optimizing quantitative metrics like high F1 scores, lower logarithmic loss values, and higher AUC for a well-calibrated, high-performing model
